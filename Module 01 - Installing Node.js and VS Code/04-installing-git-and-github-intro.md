# 📝 Text Mode (Full Explanation)

### **What Is Git?**

Git is a **version control system** — a tool that keeps track of changes in your code.

It allows you to:

* Save your work safely
* Revert mistakes
* Track progress
* Collaborate with others
* Store automation projects locally

Whenever you see developers mention “commits,” “branches,” or “pull requests,” they are talking about Git.

---

### **What Is GitHub? (Very Different from Git)**

GitHub is a **cloud platform** where your Git repositories live online.

Git = Tool on your computer
GitHub = Website for hosting your Git repositories

Think of it like this:

📦 **Git (local)** = saves your code on your machine
☁️ **GitHub (cloud)** = stores and shares your code online

You will use GitHub to upload your automation tests for:

* Assignments
* Mini-projects
* Interview portfolios
* Code reviews
* Immersive PR (Pull Request) sessions

---

### **How to Install Git**

1. Go to the official download page:
   [https://git-scm.com/downloads](https://git-scm.com/downloads)

2. Choose your OS (Windows, macOS, Linux automatically shown)

3. Download the installer

4. Run the installer:

   * Accept the license
   * Keep all **default settings**
   * Complete installation

Git installs both the Git system AND **Git Bash** (a terminal you can use if you prefer).

---

### **Verify Git Installation**

Open your terminal (or Git Bash) and run:

```
git --version
```

Expected output:

```
git version 2.xx.x
```

If you see a version number → ✔ Git is installed successfully.

---

### **Create Your GitHub Account**

1. Go to: [https://github.com](https://github.com)
2. Click **Sign Up**
3. Enter your email, password, and username
4. Verify your account
5. Log in to GitHub

You're now ready to host automation projects in your own GitHub profile.

---

### **Why Automation Engineers Use Git & GitHub**

Automation engineers rely on Git and GitHub to:

✔ Store test cases and frameworks
✔ Share automation code during interviews
✔ Submit assignments
✔ Collaborate on test suites
✔ Contribute to PR reviews
✔ Track issues and debug history

Every automation engineer should know the basics — and you are already ahead.

# 🧪 Mini Code Challenge

Open your terminal and run:

```
git --version
```

Then run:

```
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
```

This configures Git with your identity.
