# 📝 Text Mode (Full Explanation)

### **What Is Automation Testing?**

Automation testing means using **software tools** (like Playwright) to automatically check that a website or application behaves correctly.

Instead of manually:

- Opening a browser  
- Clicking buttons  
- Typing into fields  
- Checking results  

…automation does it **for you**, quickly and repeatedly.

---

### **Manual Testing vs Automation Testing**

| Manual Testing | Automation Testing |
|----------------|-------------------|
| Performed by a human | Performed by a script |
| Slow and repetitive | Fast and consistent |
| Prone to human error | Very accurate |
| Hard to repeat 100 times | Easy to repeat forever |
| Good for exploratory work | Good for checks and regressions |

Both manual and automation testing are important — but automation becomes critical when apps grow.

---

### **Why Companies Use Automation**

Companies rely on automation because it:

✔ Saves time  
✔ Reduces repetitive work  
✔ Improves coverage  
✔ Finds bugs earlier  
✔ Supports fast releases  
✔ Ensures consistent quality  

Automation is especially powerful in **CI/CD pipelines** where tests run automatically on every code change.

---

### **What Tools Are Used for Automation?**

There are many, but this program focuses on:

- **Playwright (JavaScript/TypeScript)**  
- Runs across **Chrome, Firefox, WebKit**  
- Very fast, reliable, and modern  
- Built by Microsoft  

You will complete your **first automated Playwright test** by the end of this course.

---

### **How Automation Testing Works (Simple Explanation)**

Automation tools:

1. **Launch a browser**  
2. **Open a webpage**  
3. **Find elements** (buttons, inputs, links)  
4. **Interact with them** (click, type, submit)  
5. **Check the result** (assertions)  
6. **Report success or failure**

This course teaches you *exactly* how to do these steps.
